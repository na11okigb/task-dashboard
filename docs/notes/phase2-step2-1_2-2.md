## このフェーズの背景原則

**Leaky Abstractions（漏れのある抽象化の法則）** — Joel Spolsky

ReactはHTMLを生成し、Tailwind CSSはCSSを生成する。これらの抽象化ツールが問題を起こした時、その下にあるHTML/CSSを理解していなければ原因を特定できない。だからReactやTailwindの前に、素のHTML/CSSを触る。

ただし、HTML/CSSを完璧にする必要はない。Phase 4でTailwind CSS v4を導入すれば、CSSの大半はユーティリティクラスで書くことになる。

## Step 2-1: HTMLのセマンティクス

### 核心: HTMLは「見た目」ではなく「意味」を記述する

HTMLを「読む」のは人間だけではない。スクリーンリーダー（視覚障害者が使う読み上げソフト）や検索エンジンのクローラーもHTMLの意味構造を読む。正しいタグを使うことで、アクセシビリティ（a11y）とSEOの両方に対応できる。

### セマンティック要素の分類

**構造要素（ページの骨格）:**
- `<header>` — ページやセクションのヘッダー
- `<nav>` — ナビゲーションリンクのまとまり（ページ間の移動）
- `<main>` — ページの主要コンテンツ（1ページに1つだけ）
- `<section>` — テーマでまとまったコンテンツの区画（ページの中での区切り）
- `<article>` — 独立して意味をなすコンテンツ（配信の単位になるもの）
- `<aside>` — メインコンテンツの補助情報（フィルター、関連情報など）
- `<footer>` — フッター

**テキスト要素:**
- `<h1>`〜`<h6>` — 見出しの階層（`<h1>` はページに1つが望ましい）
- `<p>` — 段落
- `<strong>` — 重要性の強調 / `<em>` — ニュアンスの強調

**意味を持たないコンテナ:**
- `<div>` — ブロックレベルの汎用コンテナ（スタイリング目的のみ）
- `<span>` — インラインレベルの汎用コンテナ

### 重要な使い分け

**section vs article:**
- `<section>` — ページの区画。そのページの構造を分けるためのもの
- `<article>` — 独立したコンテンツの単位。RSSやSlackで配信できるもの
- 判断基準: 「それ独自で機能するか？」→ Yes なら article、No なら section

**nav vs aside:**
- `<nav>` — 別の画面に遷移するためのリンク（ページ間の移動）
- `<aside>` — 今のページの補助機能（フィルター、関連情報など。なくてもメインコンテンツは成立する）

**divを使う前に:**
適切なセマンティック要素がないか考える → なければ `<div>` を使う

### その他の学び

- `lang="ja"` — ページの言語を指定。スクリーンリーダーが読み上げ言語を切り替える
- `<nav>` 内の `<li>` には `<a>` タグを使う — 「クリックできる場所」という意味をHTMLが持つ
- `<dl>/<dt>/<dd>` — 定義リスト。ラベルと値のペアを表現する（タスクカードのステータス/担当者/優先度に適切）
- `<small>` — 細則・注釈。フッターのコピーライトに適切

## Step 2-2: CSSレイアウト — Flexbox と Grid

### ボックスモデル — すべてのHTML要素の基本構造

すべてのHTML要素は内側から順に4層構造を持つ:

1. **content** — 中身そのもの（テキスト、画像など）の領域
2. **padding** — contentの周りの内側の余白
3. **border** — 要素の周りに表示される枠線
4. **margin** — 要素の外側の余白（隣の要素との距離）

**額縁の例え:**
- 写真 = content
- 白い台紙 = padding
- 額縁のフレーム = border
- 額縁と隣の額縁の間の壁 = margin

**box-sizing: border-box（ほぼ必須）:**

デフォルトでは `width: 200px` と書くとcontentだけが200pxになり、paddingとborderを足すと実際の幅が200pxより大きくなる。`box-sizing: border-box` を設定すると、widthにpaddingとborderが含まれ、直感通りの挙動になる。

```css
*, *::before, *::after {
  box-sizing: border-box;
}
```

`*` はユニバーサルセレクタ（全要素を対象にする）。

### Flexbox — 1次元レイアウト

**要素を一方向（横 or 縦）に並べて配置を制御する。**

基本: 親要素に `display: flex` をつけると、子要素が横並びになる（デフォルトは横 = row）。

覚えるプロパティは3つ:

| プロパティ | 役割 |
|-----------|------|
| `justify-content` | 主軸（並ぶ方向）の配置。余ったスペースをどう配分するか |
| `align-items` | 交差軸（主軸と直角の方向）の配置 |
| `gap` | 要素間のスペース |

**重要:** `justify-content` と `align-items` の効く方向は `flex-direction` に連動して回転する。横並び（row）なら justify=横/align=縦。縦並び（column）なら justify=縦/align=横。

**`display: flex` の本質:**
「横並びにするためのもの」ではなく、「要素の配置を自在にコントロールするモードに切り替える」もの。`flex-direction: column` にしても、`justify-content: center` で縦方向の中央揃えができるなど、デフォルトの縦積みではできない配置制御が可能になる。

### Grid — 2次元レイアウト

**縦と横の両方を同時に制御する。ページ全体の骨格を組むのに使う。**

```css
.dashboard {
  display: grid;
  grid-template-columns: 200px 1fr;
}
```

**fr単位:** 「fraction（比率）」の略。余ったスペースを比率で分け合う。
- `1fr 1fr 1fr` → 均等3分割
- `250px 1fr` → 左を250px固定、残り全部を右に（サイドバーレイアウトの定番）

**grid-template-areas:** レイアウトを「地図」のように書ける。
```css
grid-template-areas:
  "header  header"
  "sidebar main"
  "footer  footer";
```

### FlexboxとGridの使い分け

| 用途 | 選択 | 例 |
|------|------|-----|
| 部品の中の配置（1方向） | Flexbox | カード内の横並び、ナビのリンク |
| ページ全体の骨格（2方向） | Grid | サイドバー＋メイン、ヘッダー＋フッター |

### CSSの書き方のプラクティス

- **インラインスタイル（`style="..."`）は避ける** — 同じスタイルを複数箇所にコピペすることになる。CSSクラスを使えば1箇所で管理できる
- **`flex-direction: row` は省略可能** — Flexboxのデフォルトが横並び（row）なので明示不要。デフォルト値を書くとコードのノイズになる
- **`gap` でスペース管理** — marginを個別指定するより、Flexbox/Gridの `gap` で統一的に管理する
- **`list-style: none`** — `<ul>/<li>` のデフォルトの黒丸を消す（ナビゲーション等で使用）

## 要復習ポイント
- justify-content と align-items の方向が flex-direction に連動する点（主軸/交差軸）
- justify-content は「余ったスペースをどう配分するか」なので、余白がなければ効果が見えない
